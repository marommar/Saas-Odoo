Componentes del Sistema

Conexión Odoo

Consultas en tiempo real de clientes, inventario y presupuestos a través de Odoo API JSON-RPC utilizando credenciales aisladas por tenant.

Motor NLP

Implementación de modelos de lenguaje grandes (LLMs) para la orquestación y el procesamiento de intenciones de usuarios y empleados.

Persistencia

Almacenamiento de historiales de conversación y recuperación de contexto (RAG) empleando bases de datos PostgreSQL y Supabase.

Flujo de Datos (Workflow)

El sistema opera bajo un entorno de ejecución seguro (n8n). Cada solicitud entrante se procesa bajo el siguiente esquema:

1. Identificación Multi-Tenant

Extracción del tenant_id del payload para aislar entornos.

2. Recuperación de Entorno

Consulta a Supabase para recuperar la URL, BD y API Key específicos del cliente.

3. Ejecución Autónoma

El agente IA ejecuta herramientas dinámicas contra Odoo (ej. search_read sobre res.partner) y formula la respuesta final.
